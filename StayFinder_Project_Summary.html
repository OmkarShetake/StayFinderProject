01 — Overview

What is StayFinder?

StayFinder is a full-stack property rental marketplace modelled after Airbnb. Users can search for properties by city, dates, and category; hosts can list and manage properties; guests can book, review, and track stays; admins can approve or reject listings. Real-time notifications are delivered via WebSocket.

02 — Technology Stack

Built With

☕

Java 21 + Spring Boot 3

Virtual threads enabled for high concurrency. Spring Security for JWT-based stateless auth. Spring Data JPA with Hibernate for ORM.

spring-boot-starter-web
🐘

PostgreSQL 15

Relational database with Flyway migrations for schema versioning. Native SQL queries with ::text casting for safe ILIKE searches.

postgresql-42.6.0
🔐

JWT Authentication

Access tokens (24h) + refresh tokens (7d) stored in localStorage. Auto-refresh on 401 with retry logic built into the API client.

jjwt-api-0.12
🔌

WebSocket / STOMP

Real-time push notifications via SockJS + STOMP. User-specific queues at /user/{id}/queue/notifications.

spring-websocket
🌐

Vanilla JS Frontend

No framework — pure HTML, CSS, and modular JS objects (API, Auth, Utils, WS). SPA-style navigation with nginx try_files.

SockJS + Stomp.js
🐳

Docker + nginx

Three-container stack: postgres, backend, frontend. nginx proxies /api/ and /ws to Spring Boot with 3600s WS timeout.

docker-compose v3.9
03 — Features

What It Does

🔑

Authentication & Users

  • Register / Login with email + password
  • JWT access token + refresh token flow
  • Three roles: Guest, Host, Admin
  • Any Guest can "become a host" in one click
  • Superhost badge for top-rated hosts
  • Auto token refresh — no forced logouts
🏠

Property Listings

  • 9 categories: Beach, Mountain, City, Countryside, Lakefront, Unique, Heritage, Camping, All
  • 3 types: Entire Home, Private Room, Shared Room
  • 12 amenities: WiFi, AC, Kitchen, Pool, Gym, etc.
  • Multiple photos, price per night + weekend price
  • Instant Book toggle
  • Admin approval flow before listing goes live
📅

Booking System

  • Interactive calendar with availability blocking
  • Real-time price preview (base + cleaning + service fee)
  • Long-stay discount & weekend pricing support
  • Unique reference ID per booking
  • Guest can cancel; Host can confirm or reject
  • Status flow: PENDING → CONFIRMED / REJECTED / CANCELLED
⭐

Reviews & Ratings

  • 6-dimension review: Cleanliness, Communication, Check-in, Location, Value, Accuracy
  • Overall rating auto-calculated as average
  • Only guests with completed stays can review
  • Property avg rating updates after each review
  • Rating summary with per-category breakdown
🔔

Real-Time Notifications

  • WebSocket push via STOMP protocol
  • Notifications for: booking requests, confirmations, cancellations, new reviews
  • Unread badge in navbar
  • Mark all read in one click
  • Toast pop-ups for live events
❤️

Wishlist & Host Dashboard

  • Save / unsave any property with heart button
  • Dedicated wishlist page
  • Host dashboard: earnings, bookings, listings
  • Host can update property availability calendar
  • Admin panel: approve / reject pending listings
  • Scheduled job auto-completes past bookings

07 — User Flows

How It All Works

Guest Booking Flow

1

Search

Enter city, dates, guests → GET /properties/search returns paginated results filtered by availability

2

View Property

Click card → property detail page loads photos, amenities, host info, availability calendar, and reviews

3

Price Preview

Select dates → POST /bookings/price-preview returns base + cleaning + service fee breakdown

4

Reserve

POST /bookings → booking created with status PENDING (or CONFIRMED if instant book)

5

Notification

Host receives real-time WebSocket push. Guest sees booking in Trips page

6

Review

After stay completes, guest can submit 6-dimension review → property rating updates

Host Listing Flow

1

Become Host

POST /auth/become-host → user.isHost set to true, dashboard unlocked

2

Create Listing

Fill form: title, address, type, category, price, amenities, photos → POST /host/properties → status: PENDING

3

Admin Approval

Admin logs in → reviews pending listings → PATCH /admin/properties/{id}/approve

4

Goes Live

Status changes to APPROVED → property appears in search results and category pages

5

Manage Bookings

Host dashboard shows incoming requests → confirm or reject → guest notified via WebSocket

08 — Fixes Applied

All Bugs Fixed in This Version

CRITICAL

Hardcoded API base URL (api.js)

Was http://localhost:8080/api/v1 — caused CORS failures when served via nginx on port 3000. Fixed to use /api/v1 (relative) for Docker, and localhost:8080 directly for IntelliJ/dev servers.

CRITICAL

PostgreSQL lower(bytea) error (PropertyRepository.java)

JPQL LOWER(:city) passed the parameter as bytea instead of text, crashing every search. Replaced with native SQL using p.city::text explicit cast.

CRITICAL

Missing service methods (BookingService + PropertyService)

Controllers called getBookingById(), getAllBookings(), getByStatus(), and rejectProperty() — none of which existed in the service layer. Added all four with correct logic.

CRITICAL

No JWT auto-refresh (api.js)

When access token expired, every API call silently failed with 401. Now auto-refreshes using the refresh token and retries the request. Logs out only if refresh also fails.

FUNCTIONAL

Notification serialization crash (Notification.java)

The user field used FetchType.LAZY but was serialized directly (no DTO), causing LazyInitializationException and exposing password hashes. Fixed with @JsonIgnore.

FUNCTIONAL

Search button, logout, auth tabs had no event listeners (index.html)

Navbar search button had no click handler. Auth tab switching, login/register form submission, and logout events were not wired up in the home page script block.

FUNCTIONAL

Host listing form missing amenities & image URL fields (host.html)

The Add Listing form had no way to attach amenities or photos — both accepted by the backend but never sent. Added 12 amenity checkboxes and a comma-separated image URL textarea.

INFRA

WebSocket connections dropped after 60s (nginx.conf)

nginx's default proxy timeout is 60 seconds, silently closing WS connections. Fixed with proxy_read_timeout 3600s and proxy_send_timeout 3600s.

INFRA

Backend crash not recovered (docker-compose.yml)

If Spring Boot crashed on startup (DB not ready), it stayed down. Added restart: on-failure, backend health check via /actuator/health, and frontend waits for healthy backend.

INFRA

Wrong redirect paths throughout (auth.js, host.js, property.js, all pages)

All /index.html and ../index.html navigation redirects failed under nginx's SPA routing. Changed all to / and /?login=1.

09 — Getting Started

How to Run

Option A — Full Docker Stack

Runs everything: PostgreSQL + Spring Boot + nginx. Open http://localhost:3000

# one command does everything
$ docker-compose up --build

Option B — IntelliJ + Docker DB

Run only PostgreSQL in Docker; run Spring Boot in IntelliJ; serve frontend separately.

# step 1: start only the database
$ docker-compose up postgres -d

# step 2: open backend/ in IntelliJ → Run StayFinderApplication

# step 3: serve frontend
$ cd frontend && python3 -m http.server 5500

# open http://localhost:5500
